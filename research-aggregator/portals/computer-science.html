Introduction to Computer Science

Computer Science is the study of computation, algorithms, and information processing. It encompasses both theoretical foundations and practical applications, from understanding how computers work at a fundamental level to building complex software systems.

What is Computer Science?

At its core, computer science is about solving problems efficiently. Unlike other sciences that study natural phenomena, computer science is largely a constructed discipline - we study systems that humans have created and continue to create. This gives it a unique character: the laws we discover are often laws we've designed.

üí°

Key Insight

Computer science is not just about programming. Programming is a tool we use to implement solutions, but the real work is in understanding problems, designing solutions, and analyzing their efficiency.

Core Areas of Computer Science

  • Theory of Computation - What can be computed? What are the limits of computation?
  • Algorithms & Data Structures - How do we solve problems efficiently?
  • Programming Languages - How do we express computations?
  • Computer Architecture - How do computers physically work?
  • Operating Systems - How do we manage computer resources?
  • Networks & Distributed Systems - How do computers communicate?
  • Artificial Intelligence - Can computers think?
  • Software Engineering - How do we build large, reliable systems?

Prerequisites for This Guide

This guide assumes basic familiarity with using computers but no prior programming experience. We'll build from the ground up, explaining each concept as we encounter it. Mathematical concepts will be introduced as needed.

Algorithms & Data Structures Fundamentals

An algorithm is a step-by-step procedure for solving a problem. A data structure is a way of organizing data so that it can be used efficiently. Together, they form the foundation of all software.

What Makes a Good Algorithm?

A good algorithm is:

  • Correct - It produces the right answer for all valid inputs
  • Efficient - It uses minimal time and memory
  • Simple - It's easy to understand and implement
  • Robust - It handles edge cases gracefully

Big O Notation

Big O notation is how we describe algorithm efficiency. It tells us how the running time or space requirements grow as the input size grows.

Notation Name Example 10 items 1000 items
O(1) Constant Array access 1 1
O(log n) Logarithmic Binary search 3 10
O(n) Linear Simple search 10 1000
O(n log n) Linearithmic Merge sort 30 10000
O(n¬≤) Quadratic Bubble sort 100 1000000
O(2^n) Exponential Brute force 1024 Impractical

Essential Data Structures

Arrays

An array is a contiguous block of memory storing elements of the same type. Access is O(1), but insertion/deletion can be O(n).

Python 
# Array operations
arr = [1, 2, 3, 4, 5]

# Access - O(1)
element = arr[2]  # Gets 3

# Append - O(1) amortized
arr.append(6)  # [1, 2, 3, 4, 5, 6]

# Insert at position - O(n)
arr.insert(0, 0)  # [0, 1, 2, 3, 4, 5, 6]

# Search - O(n)
index = arr.index(3)  # Returns 3

Linked Lists

A linked list stores elements in nodes, each pointing to the next. Insertion/deletion is O(1) at known positions, but access is O(n).

Python 
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node

    def prepend(self, data):  # O(1)
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node

Hash Tables

A hash table provides O(1) average-case access, insertion, and deletion by using a hash function to map keys to array indices.

When to Use Hash Tables

Use hash tables when you need fast lookups by key, counting occurrences, or detecting duplicates. They're the go-to data structure for many interview problems.

Trees

A tree is a hierarchical structure with a root node and child nodes. Binary Search Trees (BSTs) maintain sorted order, enabling O(log n) operations on balanced trees.

Graphs

A graph consists of vertices (nodes) and edges (connections). Graphs model relationships: social networks, road maps, dependencies, and more.

Programming Paradigms

A programming paradigm is a fundamental style of programming that determines how we structure and organize code. Understanding different paradigms makes you a more versatile programmer.

Imperative Programming

Imperative programming tells the computer exactly what steps to take. You write sequences of statements that change program state.

Python - Imperative Style 
# Imperative: explicit step-by-step instructions
def sum_squares_imperative(numbers):
    result = 0
    for n in numbers:
        squared = n * n
        result = result + squared
    return result

Functional Programming

Functional programming treats computation as mathematical function evaluation. It emphasizes immutability and avoiding side effects.

Python - Functional Style 
# Functional: describe what, not how
def sum_squares_functional(numbers):
    return sum(map(lambda x: x * x, numbers))

# Or using list comprehension
def sum_squares_comprehension(numbers):
    return sum(x * x for x in numbers)

Object-Oriented Programming

Object-Oriented Programming (OOP) organizes code around objects that combine data and behavior. Key concepts include encapsulation, inheritance, and polymorphism.

Python - OOP Style 
class Shape:
    """Base class for shapes - demonstrates encapsulation"""
    def __init__(self, color):
        self._color = color  # Protected attribute

    def area(self):
        raise NotImplementedError

    def describe(self):
        return f"A {self._color} shape with area {self.area()}"

class Rectangle(Shape):
    """Inheritance: Rectangle is a Shape"""
    def __init__(self, color, width, height):
        super().__init__(color)
        self.width = width
        self.height = height

    def area(self):  # Polymorphism: different shapes calculate area differently
        return self.width * self.height

class Circle(Shape):
    def __init__(self, color, radius):
        super().__init__(color)
        self.radius = radius

    def area(self):
        import math
        return math.pi * self.radius ** 2

# Usage - polymorphism in action
shapes = [Rectangle("red", 4, 5), Circle("blue", 3)]
for shape in shapes:
    print(shape.describe())  # Same method call, different behavior

OOP Pitfalls

Avoid deep inheritance hierarchies. Favor composition over inheritance when possible. Not everything needs to be a class - sometimes a function is simpler and clearer.

Software Engineering Principles

Software engineering is about building software that works, is maintainable, and can evolve over time. These principles guide professional software development.

SOLID Principles

The SOLID principles are five design principles for object-oriented software:

  • S - Single Responsibility: A class should have only one reason to change
  • O - Open/Closed: Open for extension, closed for modification
  • L - Liskov Substitution: Subtypes must be substitutable for their base types
  • I - Interface Segregation: Many specific interfaces are better than one general interface
  • D - Dependency Inversion: Depend on abstractions, not concretions

Design Patterns

Design patterns are reusable solutions to common problems. Here are essential patterns every developer should know:

Singleton Pattern

Ensures a class has only one instance. Useful for configurations, connection pools, or logging.

Python 
class DatabaseConnection:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance.connection = cls._create_connection()
        return cls._instance

    @staticmethod
    def _create_connection():
        return "Database connection established"

# Both variables reference the same instance
db1 = DatabaseConnection()
db2 = DatabaseConnection()
print(db1 is db2)  # True

Factory Pattern

Creates objects without specifying the exact class. Useful when object creation is complex or depends on configuration.

Observer Pattern

Defines a subscription mechanism to notify multiple objects about events. Foundation of event-driven programming.

Databases & Data Management

Databases are systems for storing, retrieving, and managing data. Choosing the right database and using it effectively is crucial for application performance.

Relational Databases (SQL)

SQL (Structured Query Language) databases organize data into tables with relationships between them. They ensure data integrity through ACID properties.

SQL 
-- Create tables with relationships
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    content TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Query with JOIN
SELECT users.username, posts.title, posts.created_at
FROM posts
JOIN users ON posts.user_id = users.id
WHERE users.username = 'john_doe'
ORDER BY posts.created_at DESC;

-- Aggregation
SELECT users.username, COUNT(posts.id) as post_count
FROM users
LEFT JOIN posts ON users.id = posts.user_id
GROUP BY users.id
HAVING COUNT(posts.id) > 5;

NoSQL Databases

NoSQL databases provide flexible schemas and horizontal scaling. Types include document stores (MongoDB), key-value stores (Redis), and graph databases (Neo4j).

Database Optimization

Key optimization techniques:

  • Indexing - Create indexes on frequently queried columns
  • Query optimization - Use EXPLAIN to analyze query plans
  • Normalization - Reduce redundancy (but denormalize for read performance)
  • Connection pooling - Reuse database connections
  • Caching - Cache frequently accessed data

System Design Fundamentals

System design is the process of defining the architecture, components, and interfaces of a system to satisfy specified requirements. It's essential for building scalable applications.

Scalability Concepts

Scalability is a system's ability to handle increased load. Two approaches:

  • Vertical Scaling (Scale Up) - Add more resources to a single machine
  • Horizontal Scaling (Scale Out) - Add more machines to distribute load

Load Balancing

A load balancer distributes incoming requests across multiple servers. Common algorithms include round-robin, least connections, and IP hash.

Caching Strategies

Caching stores frequently accessed data in fast storage. Cache strategies:

  • Cache-aside - Application checks cache first, loads from DB if miss
  • Write-through - Write to cache and DB simultaneously
  • Write-behind - Write to cache, async write to DB

Microservices Architecture

Microservices break applications into small, independent services that communicate via APIs. Benefits include independent deployment, technology flexibility, and fault isolation.

Microservices Complexity

Microservices add operational complexity: distributed transactions, service discovery, network latency. Start with a monolith and extract services as needed.

Artificial Intelligence & Machine Learning

Artificial Intelligence (AI) enables machines to perform tasks that typically require human intelligence. Machine Learning (ML) is a subset where systems learn from data.

Types of Machine Learning

Supervised Learning

Learn from labeled examples. Given input-output pairs, predict outputs for new inputs.

  • Classification - Predict categories (spam/not spam)
  • Regression - Predict continuous values (house prices)

Unsupervised Learning

Find patterns in unlabeled data.

  • Clustering - Group similar items (customer segments)
  • Dimensionality Reduction - Compress features while preserving information

Reinforcement Learning

Learn by interacting with an environment to maximize rewards. Used in game AI and robotics.

Neural Networks & Deep Learning

Neural networks are computing systems inspired by biological brains. Deep learning uses neural networks with many layers to learn complex patterns.

Python - Simple Neural Network 
import numpy as np

# Simple 2-layer neural network
class NeuralNetwork:
    def __init__(self, input_size, hidden_size, output_size):
        # Initialize weights randomly
        self.W1 = np.random.randn(input_size, hidden_size) * 0.01
        self.W2 = np.random.randn(hidden_size, output_size) * 0.01

    def sigmoid(self, x):
        return 1 / (1 + np.exp(-x))

    def forward(self, X):
        # Forward propagation
        self.z1 = np.dot(X, self.W1)
        self.a1 = self.sigmoid(self.z1)
        self.z2 = np.dot(self.a1, self.W2)
        output = self.sigmoid(self.z2)
        return output

# Usage
nn = NeuralNetwork(3, 4, 1)
input_data = np.array([[1, 0, 1]])
prediction = nn.forward(input_data)
